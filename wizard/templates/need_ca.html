Secure communications will require certificates

Secured communications will require certificates and keys. For example the LDAP server (which is the only container that is mandatory) will require one.

Here you will have two alternatives:

  • If you are testing this software or do not care about warnings about self-signed certificates, you will create your own Certificate Authority. Check the instructions below.
  • You can use certificates from a recognized certificate authority. This is a longer process that might cost money. Check, for example: Let's encrypt.
